# Sofia's Endless Runner — Documentation

The design & engineering source-of-truth for the project. Read in this order:

| Doc | What it answers | Audience |
|-----|-----------------|----------|
| **[GDD.md](GDD.md)** | *What* the game is and *why* — vision, pillars, mechanics, worlds, economy, scope. | Design decisions |
| **[TDD.md](TDD.md)** | *How* the engine is built and should evolve — architecture, decisions, migration plan. | Implementation |
| **[ROADMAP.md](ROADMAP.md)** | *In what order* — prioritized milestones tying design to architecture. | Sequencing |

**Core idea:** this is an **engine-first** project. *Sofia's Endless Runner* is
the flagship title and proving ground for a reliable, extensible runner engine
meant to spawn visual reskins and genre mixes cheaply. Subway Surfers is the
design north star; the audience includes young children (no scary content, never
pay-to-win).

These are **living documents** — update them as decisions change. Conventions used
throughout: **Built / As-is** (exists today) · **Planned / Target** (agreed, not
built) · **Idea** (under consideration).
